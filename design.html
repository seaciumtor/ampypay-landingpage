01 · Color

One blue. Two navies. A cream that warms everything.

06

06 · Accessibility

Decoration is hidden; meaning is labeled

The flash never costs the fundamentals. Canvases, chips, and glows are aria-hidden; everything that carries meaning gets a name, a label, or a paired indicator.

Semantic landmarks

header, main, section, nav, footer; a single h1; anchor-linked section IDs.

Never color alone

Status meaning always pairs a dot with a text label — blockers, warnings, anomalies.

Modal discipline

role="dialog", aria-modal, labelled title, focus moves in, Escape and backdrop close, body scroll locks.

State stays in sync

aria-expanded / aria-hidden track the burger menu; icon-only controls carry aria-label.

Visible focus

2px blue outline with 2px offset on interactive elements — try tabbing through this page.

Descriptive alt text

Every product screenshot describes what the UI actually shows — not "dashboard screenshot."

07 · Voice

Headlines are claims, not labels

Enterprise-confident, precise, never playful. Eyebrows name the capability area; headlines make the argument; footnotes summarize the benefit with "Built to…" / "Designed to…".

why
"Built for payroll — not adapted to it"
platform
"Every payroll issue, visible before submission"
confidence
"Know your payroll is ready — before you submit"
tagline
"We handle payroll. You handle growth."

Patterns

Do

Card copy: one bold noun-phrase title + one sentence.

Mark AI features consistently with the ✦ spark + "AI" tag.

Close sections with a "Designed to…" benefit footnote.

Don't

Write headlines that just name the section ("Features", "Security").

Make claims without the footer's forward-looking disclaimer in place.

Use playful fintech tone — this is serious infrastructure.

08 · Files

No build step. No modules. No CDNs.

Plain HTML, CSS, and an IIFE. Libraries are vendored. Class names are kebab-case, describe components, and carry section prefixes.

ampypay-landingpage/ ├── index.html all markup — single page ├── design.md this system, in prose ├── css/style.css tokens at top, full component system ├── js/main.js all interaction & motion (IIFE) │ ├── world-land.js Natural Earth polygons for the globe │ └── vendor/ gsap · ScrollTrigger · lenis · three ├── assets/ │ ├── fonts/ Inter woff2 subsets, self-hosted │ ├── logos/ logo.svg · logo-white.svg · icon.svg │ ├── images/ product screenshots — png + webp │ └── misc/ flags, datacenter & provider logos ├── backend/ Node demo-form API + admin (Render) └── server.py Python stdlib fallback backend

Conventions

kebab-case component class names section prefixes: exc- · conf- · comp- · sec- · cta- · demo- no ES modules no package.json on the frontend vendored libraries only tokens defined once in :root
